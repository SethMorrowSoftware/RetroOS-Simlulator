# Required / Recommended Media for `autoexec.retro`

This campaign works without custom assets, but the expanded narrative is designed around short media clips.

## Placement

- `C:/Users/User/Videos/erebus_signal_loop.mp4`
- `C:/Users/User/Videos/webb_last_session.mp4`
- `C:/Users/User/Music/archive_hum_47.mp3`
- `C:/Users/User/Music/relay_tape_a.wav`

The script also writes an in-world copy at:
- `C:/Users/User/Documents/.archive/media/MEDIA_CATALOG.txt`

## Duration and format guidance

- Keep each file **15–40 seconds**.
- Video: H.264 MP4, 720p is sufficient.
- Audio: MP3/WAV, mono or stereo, 44.1kHz preferred.

## Creative direction

### `erebus_signal_loop.mp4`
- Abstract CRT noise, terminal glow, slight frame tears.
- No hard cuts; slow pulse rhythm.
- Should feel like a system trying to remember a face.

### `webb_last_session.mp4`
- POV desktop capture style.
- Cursor hesitation, opened notes, faint status flicker at `03:47`.
- Final frame should hold long enough to reward observation.

### `archive_hum_47.mp3`
- Low mechanical room tone + soft synth drone.
- Subtle 47-beat motif or recurring pulse.
- Avoid loud transients; this is tension bed audio.

### `relay_tape_a.wav`
- Distressed voice memo texture, light tape hiss.
- Short spoken fragments with dropouts and static bloom.
- Leave a persistent low under-tone for analysis flavor.

## Why these files matter

- THREAD-5/6 feel substantially more cinematic with intentional playback assets.
- The campaign's “observation vs. interpretation” theme lands better when players compare visual and audio channels directly.
